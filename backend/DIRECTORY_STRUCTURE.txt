WHAT EACH FILE DOES:

ocr_service.py (~250 lines)
├─ Image preprocessing (grayscale, contrast, sharpness)
├─ pytesseract integration for images
├─ pdfplumber integration for PDFs
└─ Intelligent extraction returns clean text

gemini_service.py (~280 lines)
├─ Gemini API client initialization
├─ Generic ask_gemini() function
├─ Lab report analyzer prompt
├─ Food parser prompt
├─ Symptom analysis prompt with context
└─ JSON response extraction & error handling

storage_service.py (~180 lines)
├─ Firebase Storage bucket operations
├─ upload_file() - saves files with path management
├─ download_file() - retrieves files
├─ get_public_url() - converts gs:// to https://
└─ delete_file() - cleanup

reports.py (~200 lines) - routers/
├─ POST /api/reports/upload - lab report analysis
├─ GET /api/reports/list - user's reports
├─ GET /api/reports/{id} - specific report
├─ POST /api/records/upload - health records
├─ GET /api/records/list - user's records
└─ DELETE /api/records/{id} - delete record

food.py (~150 lines) - routers/
├─ POST /api/food/log - log meal with calories
├─ GET /api/food/today - today's summary
├─ GET /api/food/history/{days} - historical
└─ POST /api/food/estimate - quick check

chat.py (~150 lines) - routers/
├─ POST /api/chat/symptom - symptom analysis
├─ POST /api/chat/quick-check - no-auth quick check
└─ GET /api/chat/history - chat history

main.py (~150 lines)
├─ FastAPI app creation
├─ CORS configuration
├─ All router registration
├─ Error handlers
├─ Health check endpoints
└─ Development server startup
